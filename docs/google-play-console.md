## 3. Privacy Policy

> **Play Console path**: Policy → App content → Privacy policy

### Privacy policy URL

```
https://jctm.org.ng/privacy-policy
```

### Requirements checklist

- [x] URL is publicly accessible (no login required)
- [x] URL uses HTTPS
- [x] Policy covers all data types collected by the app (account, diagnostics, push token)
- [x] Policy identifies the data controller: Jesus Christ Temple Ministry, Nigeria
- [x] Policy explains data retention and deletion rights
- [x] Policy describes third-party processors (Expo/FCM for push, payment processors for donations)
- [x] Policy is written in English (matches app default language)
- [x] Policy version date is visible and current

### Minimum policy content (if the hosted policy needs to be updated)

The privacy policy at the URL above must include the following disclosures:

**Data collected**:
- Email address and display name (account registration, optional)
- Viewing history and saved sermons (app functionality, on-device only unless signed in)
- Push notification token (broadcast alerts, via Expo / Firebase Cloud Messaging)
- Crash and diagnostic logs (app stability, sent to first-party API endpoint only)

**Data NOT collected**:
- Precise or coarse device location
- Contacts or address book
- Microphone or camera input
- Photos or media files
- Financial information (processed entirely by certified third-party payment partners)
- Health or fitness data
- Web browsing history

**Third-party processors**:
- Expo (EAS / Expo Push Notification Service) — push delivery infrastructure
- Firebase Cloud Messaging (FCM) — Android push delivery
- Payment processors (Paystack, Flutterwave) — donation processing only; no card data touches our servers

**User rights**:
- Account deletion available in-app: Settings → Delete Account
- Data portability: users may request an export by emailing support@templetv.org.ng
- Policy contact: privacy@jctm.org.ng

> **Play Console path**: Release → Setup

> If bandwidth costs require a phased rollout, prioritize: Nigeria, Ghana, UK, USA, Canada, South Africa.

### Managed publishing (release track workflow)

| Track | Purpose | Release status |
|---|---|---|
| **Internal testing** | Dev team validation | Active — use for every build before promoting |
| **Closed testing (Alpha)** | Ministry staff + beta testers | Active — promote from internal after smoke test |
| **Open testing (Beta)** | Public beta | Optional — use for pre-launch period |
| **Production** | Full public release | Final — promote from closed testing after approval |

**EAS submit config** (already set in `eas.json`):
```json
{
  "submit": {
    "production": {
      "android": {
        "serviceAccountKeyPath": "./google-service-account.json",
        "track": "internal",
        "releaseStatus": "completed",
        "changesNotSentForReview": true
      }
    }
  }
}
```

### Rollout strategy

For the initial production release, use a **staged rollout**:

1. Start at **10%** rollout
2. Monitor crash rate (target < 1.5%) and ANR rate (target < 0.47%) for 48 hours via Android Vitals
3. Expand to **50%** if metrics are healthy
4. Expand to **100%** after 72 hours of stable metrics

### Content guidelines

- **Sensitive permissions**: All sensitive permission uses are documented in Section 8 (Data Safety)
- **Background activity**: `FOREGROUND_SERVICE_MEDIA_PLAYBACK` is used solely for background audio/video playback (Radio Mode, live broadcast) — fully disclosed in the Play Store listing
- **App updates**: OTA JS-only updates use Expo EAS Update (`ota-update.yml`) and do not require a Play Store review cycle for JS changes

### Android TV / Lean-back track

The `androidtv` EAS profile builds a separate `.aab` for the **Android TV** track in Play Console. Configure a separate release track for this:

- Track: **Android TV — Production**
- Managed separately from the phone/tablet track
- Uses the same package ID `com.templetv.jctm` with `LEANBACK_LAUNCHER` intent filter

### Google Play app signing

**→ Use Google Play App Signing (opt-in — recommended).**

Upload your app signing key to Google's infrastructure for maximum key protection. The upload key (used with EAS) is separate from the app signing key managed by Google.
