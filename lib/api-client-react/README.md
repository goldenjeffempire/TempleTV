# `@workspace/api-client-react` — Generated React Query hooks

React Query hooks plus a typed `fetch` wrapper for every Temple TV API
endpoint. Generated by **Orval** from
[`@workspace/api-spec`](../api-spec/README.md). Consumed by the admin and TV
apps; the mobile app uses its own thin fetch layer for finer control.

---

## 1. Usage

```tsx
import { useGetLiveStatus, useListAdminVideos } from "@workspace/api-client-react";

function Dashboard() {
  const { data: live } = useGetLiveStatus();
  const { data: videos, isLoading } = useListAdminVideos({ page: 1, limit: 20 });

  if (isLoading) return <Spinner />;
  return <VideoTable videos={videos.items} liveNow={live?.isLive} />;
}
```

Provide a `QueryClientProvider` at the root of your app. The admin and TV
apps already do this in `App.tsx` / `main.tsx`.

---

## 2. The `customFetch` wrapper

`src/custom-fetch.ts` is the single fetch entry point used by every generated
hook. Customise it once and every request inherits the change. By default it:

- Reads the **admin token** from `localStorage` (admin app) and adds
  `Authorization: Bearer <token>` to admin endpoints.
- Surfaces non-2xx responses as thrown errors so React Query's `isError`
  branch fires.
- JSON-encodes / decodes automatically.

---

## 3. Source layout

```
lib/api-client-react/
├── package.json
├── tsconfig.json
└── src/
    ├── index.ts            ← re-exports the generated barrel
    ├── custom-fetch.ts     ← single fetch entry-point (customisable)
    └── generated/          ← Orval output — DO NOT EDIT BY HAND
```

The package's `exports` map points directly at `src/index.ts`, so consumers
get TypeScript source — no build step is required.

---

## 4. Regenerating

```bash
pnpm --filter @workspace/api-spec run codegen
```

This regenerates `src/generated/**` from `lib/api-spec/openapi.yaml`. Hooks
are named after each operationId in the spec
(`useGetLiveStatus`, `useUpdateAdminVideo`, ...).
