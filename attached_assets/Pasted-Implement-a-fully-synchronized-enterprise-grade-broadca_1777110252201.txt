Implement a fully synchronized, enterprise-grade broadcast behavior system for TempleTV that enforces automatic playback, strict radio/video mode separation, and real-time cross-device synchronization similar to a professional television station.

Objective:

Redesign the playback logic so that TempleTV operates as a continuous, always-on live broadcast system with no manual play/pause controls, fully synchronized audio-visual delivery, and consistent behavior across all devices.

Core Playback Behavior Requirements:

1. Remove Manual Playback Controls:

* Completely remove all play and pause controls from:

  * Live broadcast video player
  * Hero Cinematic section
  * TV Player page
  * Radio Mode interface
* Enforce automatic playback behavior where:

  * Clicking “Watch Temple TV” immediately starts the live broadcast automatically
  * No user interaction is required to start playback
  * Playback cannot be manually paused or stopped via UI controls
* Ensure playback is governed by system broadcast state, not user-driven media controls.

2. Radio Mode Behavior Rules:

* Radio Mode must function as a strict toggle-based system:

  * When Radio Mode is ON:

    * Video playback must be completely hidden or disabled
    * Only audio stream continues playing
    * Video rendering must stop immediately
  * When Radio Mode is OFF:

    * Audio-only stream stops
    * Video broadcast resumes immediately from the same live broadcast timeline
* Ensure seamless switching between audio and video without desynchronization or restart delays.

3. Live Broadcast Continuous Playback:

* Ensure all live broadcasts automatically play without interruption once initiated.
* Playback must continue indefinitely unless:

  * The user explicitly exits the platform
  * The broadcast stream itself changes at the server level
* Prevent any pause, stop, or manual interruption from the UI layer.

4. Cross-Device Broadcast Synchronization:

* Ensure all devices (mobile, web, TV, tablet) display and play the exact same live broadcast stream in real time.
* All users must be synchronized to the same ON AIR broadcast timeline.
* Even on page refresh:

  * Playback must resume automatically at the correct live broadcast position
  * No manual play interaction should be required
* Prevent drift between devices so all users experience identical real-time broadcast behavior.

5. Video and Audio Synchronization Rules:

* Ensure video and audio remain perfectly aligned across all platforms.
* Maintain a single authoritative broadcast source controlling:

  * Video stream
  * Audio stream
  * Radio Mode output
* Prevent mismatched playback states between audio and video layers.

6. UI/UX Behavior Adjustments:

* Remove all visible playback UI elements such as:

  * Play button
  * Pause button
  * Center play overlays
  * Video control bars that interrupt broadcast experience
* Maintain a clean, television-style viewing interface with no interactive media controls.
* Ensure the platform feels like a real TV channel where content is always playing.

7. Page Refresh & Session Persistence:

* Ensure live broadcast continues automatically after page reload or app restart.
* Maintain session-based synchronization with the live ON AIR timeline.
* Prevent reset of playback state or loss of broadcast position during refresh events.

8. Enterprise Broadcast Consistency:

* Implement centralized broadcast state management so all devices share:

  * Same playback time
  * Same stream source
  * Same ON AIR status
* Ensure perfect synchronization across all users globally.

Expected Final Outcome:

Deliver a fully automated, enterprise-grade television broadcasting system where:

* All live streams play automatically with no play/pause controls
* Radio Mode cleanly switches between audio-only and full video broadcast
* All devices are perfectly synchronized to the same live ON AIR stream
* Page refreshes do not interrupt playback continuity
* The platform behaves exactly like a real-world TV station broadcast system

“Temple TV Anywhere You Go”
