Execute a full platform-wide domain migration across the entire TempleTV ecosystem to replace all legacy device-link references and ensure complete consistency, stability, and production-safe operation under the new official domain structure.

Objective:

Migrate every instance of the old device-link domain:

templetv.app/link

to the new official production domain:

templetv.org.ng/link

while ensuring zero broken flows, zero legacy references, and full backward-compatible routing.

Core Migration Requirements:

Global Codebase Refactor:

* Perform a complete search-and-replace across the entire codebase and all system layers.
* Update all occurrences of:

  * Frontend UI components
  * TV application interfaces
  * Authentication and device pairing flows
  * API responses and request handlers
  * Email and SMS notification templates
  * Deep linking and URL generation logic
  * Environment configuration files
  * SEO metadata and structured data
  * Static assets and embedded references
  * Documentation and internal configuration files
* Replace all instances of:

  * `templetv.app/link`
    with:
  * `https://templetv.org.ng/link`

TV Device Pairing Screen Update:

* Update all TV sign-in and device pairing interfaces across all platforms.
* Ensure the displayed instruction consistently reads:

  * “Open templetv.org.ng/link and enter the code below”
* Validate changes across:

  * Smart TV apps
  * Web-based TV interfaces
  * Mobile-to-TV pairing screens
  * Fallback HTML templates
  * Localization and translated strings

Environment Configuration Update:

* Update all environment variables across staging and production environments:

  * NEXT_PUBLIC_DEVICE_LINK_URL
  * DEVICE_LINK_URL
  * APP_BASE_URL
* Set all values to:

  * `https://templetv.org.ng/link`
* Remove all legacy references to the old domain.

Backend System Updates:

* Update all backend logic that generates or validates device links:

  * Device activation endpoints
  * OAuth/device-code verification flows
  * Redirect generation logic
  * Deep link construction services
* Ensure all dynamically generated URLs now consistently use:

  * `https://templetv.org.ng`

Critical Redirect Safety Implementation:

* Implement a permanent HTTP 301 redirect:

  * `templetv.app/link → templetv.org.ng/link`
* Ensure:

  * No broken TV pairing sessions
  * No interruption to active user flows
  * SEO authority is preserved
  * Legacy links remain functional through redirection
* Ensure redirect logic is fast, reliable, and globally distributed via edge/CDN layers.

Cache & Deployment Operations:

* Clear all CDN caches and edge caches.
* Invalidate outdated static builds and routing tables.
* Rebuild frontend applications across all platforms.
* Restart backend services to apply updated configurations.
* Regenerate all static and dynamic pages containing legacy references.

Automated Validation & Safety Checks:

Implement strict validation rules to ensure:

* No remaining references to `templetv.app/link` exist anywhere in the system.
* All TV pairing screens correctly display the new domain.
* QR codes resolve to the correct updated URL.
* Email and SMS templates contain updated links.
* Device authentication and activation flows function correctly end-to-end.
* No broken redirects or mismatched routing behaviors exist.

Fail the build or deployment pipeline if any legacy domain reference is detected.

Acceptance Criteria:

* 100% removal of all `templetv.app/link` references across the entire platform.
* Full operational functionality using `templetv.org.ng/link`.
* Seamless device pairing and authentication flow.
* Successful backward compatibility through secure 301 redirects.
* Verified stability across all platforms: web, mobile, TV, and backend services.

Execution Mode:

* FULL REFACTOR MODE
* SAFE MIGRATION PROTOCOL
* ZERO-REGRESSION ENFORCEMENT
* AUTO-VALIDATION ENABLED

Expected Final Outcome:

A fully migrated, enterprise-grade, production-safe TempleTV ecosystem operating exclusively on the new official domain with complete consistency across all platforms, zero legacy dependency risks, and fully stable device-link functionality.
