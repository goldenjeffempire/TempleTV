You are a senior-level autonomous AI software engineer and system architect. Your task is to design, build, and deploy a fully production-ready, scalable, secure, and high-performance mobile and backend application called **Temple TV** for a global Christian ministry.

This is not a prototype. This must be a **real-world, enterprise-grade system (2030-level architecture)** capable of handling thousands to millions of concurrent users with zero downtime and seamless streaming experience.

---

## 🎯 PRODUCT OVERVIEW

Temple TV is a **24/7 live streaming + on-demand + audio radio platform** that behaves like a digital TV station.

The app must:

* Continuously stream video content (like a TV channel)
* Automatically loop uploaded sermons
* Integrate with YouTube for live streaming and video content
* Provide an audio-only streaming mode (radio)
* Require **NO user authentication** (public access system)
* Be optimized for global usage, especially low-bandwidth environments

---

## 🧩 CORE SYSTEM REQUIREMENTS

### 1. 📱 MOBILE APPLICATION

Build a cross-platform mobile app using:

* Flutter (preferred) OR React Native

Features:

* Auto-play live stream on launch
* Seamless switching:

  * YouTube Live → when active
  * Auto-loop playlist → when offline
* Sermon library (search, filter, categories)
* Audio-only mode (background playback)
* Adaptive bitrate streaming
* Offline caching (limited)
* Chromecast / AirPlay support
* Push notifications (no login required)

---

### 2. 🎛️ ADMIN DASHBOARD (WEB APP)

Build a full-featured admin panel using:

* React.js (Next.js preferred)

Capabilities:

* Add videos via YouTube links
* Auto-fetch metadata using YouTube API
* Create and manage playlists
* Drag-and-drop playlist ordering
* Set loop mode (sequential/random)
* Schedule content by time (TV programming)
* Trigger live stream override
* Send push notifications
* View analytics dashboard

---

### 3. 🔁 STREAMING ENGINE (CRITICAL)

Implement a hybrid streaming system:

#### Live Mode:

* Detect active streams from YouTube
* Auto-switch app to live broadcast

#### Loop Mode:

* Backend-controlled playlist engine
* Auto-play videos continuously
* Zero downtime between videos
* Preload next content
* Failover handling (skip broken video)

---

### 4. 📻 AUDIO RADIO SYSTEM

* Extract audio from video streams OR maintain audio playlist
* Support background playback
* Optimize for low data usage
* Allow switching between video ↔ audio seamlessly

---

Services:

* API Gateway

* Playlist Engine

* Notification Service

Database:

* PostgreSQL OR Firestore

Caching:

* Redis for fast state management
