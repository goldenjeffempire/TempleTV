Implement a full enterprise-grade broadcast UX and system refinement across all TempleTV distribution platforms to standardize live playback presentation, remove time-based media indicators, and ensure production-ready consistency across the entire ecosystem.

Objective:

Refactor the broadcasting interface and underlying playback logic so that TempleTV operates strictly as a real-time live television system, removing all duration-based indicators and enforcing a unified “LIVE broadcast” experience across all platforms and devices.

Core Requirements:

1. Remove Time/Durations From Broadcast UI:

* Completely remove all time-related indicators from:

  * Live broadcast player
  * Hero Cinematic section
  * TV Player page
  * Broadcast overlays and UI elements
* Eliminate display of:

  * Video duration
  * Remaining time
  * Elapsed minutes
  * Countdown timers
  * Audio length indicators

2. Replace With “LIVE” Broadcast Format:

* Replace all time-based badges and counters with a single unified live indicator:

  * “LIVE”
* Ensure the LIVE badge:

  * Is consistently displayed across all platforms
  * Reflects real-time broadcast status only
  * Updates dynamically based on ON AIR state
* Ensure no temporal metrics are exposed to users, reinforcing a true television broadcast experience.

3. Broadcast Experience Standardization:

* Ensure all broadcast interfaces operate like a professional TV station feed:

  * Continuous streaming
  * No visible media duration metadata
  * No progress bars tied to time tracking
* Maintain consistent UI behavior across:

  * Mobile applications
  * Web platform
  * Smart TV applications
  * Tablet interfaces

4. Enterprise-Grade Broadcasting Consistency:

* Ensure all broadcast features and functionalities are fully operational, stable, and production-ready across the entire distribution ecosystem.
* Standardize playback behavior so all platforms:

  * Reflect the same ON AIR status
  * Display identical LIVE indicators
  * Maintain synchronized streaming states
* Prevent inconsistencies in broadcast UI or timing-related metadata between platforms.

5. System-Wide Synchronization Requirements:

* Ensure broadcast state management is unified across all platforms:

  * ON AIR status
  * LIVE indicator state
  * Radio Mode state
* Maintain real-time consistency across all devices and sessions.

6. UX / TV Station Experience Alignment:

* Ensure the platform visually and functionally behaves like a real-world television station:

  * No time-based UI elements
  * Continuous live broadcast feel
  * Minimal metadata exposure
  * Clean, immersive viewing experience
* Ensure all broadcast interfaces reinforce a “real TV channel” perception.

Expected Final Outcome:

Deliver a fully standardized, enterprise-grade broadcasting system where:

* All time-based media indicators are completely removed
* All broadcasts display a unified “LIVE” status instead of durations
* The entire platform operates like a real television station with continuous, real-time streaming
* All distribution platforms remain fully synchronized, consistent, and production-ready
