Refactor and strengthen the authentication system to ensure a secure, persistent, and enterprise-grade session management flow across all user interactions.

Requirements:

* Once a user is successfully authenticated, the session must remain active and persistent across page reloads, app restarts, and normal usage flows until explicitly terminated.
* Authentication state should only be invalidated when:

  * The user explicitly logs out, or
  * The user accesses the platform from a new or unrecognized device (based on device fingerprinting or secure device ID tracking policies).
* Implement secure token-based authentication (e.g., access token + refresh token mechanism) with automatic token renewal to prevent unnecessary logouts during active sessions.
* Ensure persistent session storage using secure storage mechanisms appropriate to the platform (HTTP-only cookies for web or secure storage for mobile/desktop clients).
* Introduce device-aware session management so that trusted devices maintain long-lived authentication while new devices trigger re-authentication or verification workflows.
* Prevent session loss during network interruptions, app backgrounding, or temporary service downtime through robust session recovery logic.
* Strengthen security using encryption, token rotation, session invalidation controls, and anomaly detection for suspicious login behavior.
* Ensure compliance with enterprise-grade security standards, including protection against session hijacking, replay attacks, and unauthorized access.
* Maintain seamless user experience so authenticated users do not need to repeatedly log in unless a security or logout event occurs.
* Ensure scalability and reliability of the authentication service under high traffic and multi-device usage scenarios.

The final system should deliver a secure, persistent, and frictionless authentication experience that is resilient, production-ready, and aligned with enterprise-level identity and access management standards.
