Optimize and refactor the Broadcast Queue video upload system to deliver maximum upload performance, adaptive transfer efficiency, and enterprise-grade reliability based on the user’s available device and network capabilities.

Requirements:

* Significantly improve video upload speed within the Broadcast Queue system to ensure uploads occur as fast and efficiently as the user’s device hardware, bandwidth, and network conditions allow.
* Implement intelligent adaptive upload optimization that dynamically adjusts upload behavior based on:

  * Network speed
  * Connection stability
  * Device performance
  * File size
  * Available bandwidth
* Support high-performance upload mechanisms such as:

  * Chunked uploads
  * Parallel uploads
  * Resumable uploads
  * Background upload processing
  * Streamed upload handling for large media files
* Prevent upload freezing, stalling, timeout failures, or unnecessary buffering during large file transfers.
* Implement automatic retry and recovery handling for interrupted or unstable network connections.
* Ensure upload progress updates occur in real time with accurate percentage tracking, upload speed indicators, estimated remaining time, and upload status notifications.
* Optimize backend processing pipelines to minimize delays between upload completion and media availability inside the Broadcast Queue.
* Reduce unnecessary server-side processing bottlenecks that slow down upload finalization or media indexing.
* Ensure uploaded videos become immediately available for:

  * Queue insertion
  * Instant broadcasting
  * Scheduled broadcasting
  * Playback validation
* Implement efficient file handling and media processing workflows to support very large video uploads in production environments.
* Optimize storage write operations, database synchronization, and media indexing for high-speed performance.
* Ensure compatibility across desktop systems, mobile devices, tablets, and smart administrative interfaces.
* Add enterprise-grade monitoring, upload logging, failure diagnostics, and performance analytics for upload operations.
* Strengthen upload security validation while maintaining maximum performance efficiency.
* Ensure the entire upload architecture is scalable, fault-tolerant, resilient, and production-ready for high-volume broadcasting operations.

The final implementation should provide an ultra-fast, stable, intelligent, and enterprise-grade upload experience where videos upload at the highest possible speed permitted by the user’s network and device performance while remaining reliable under real-world production conditions.
